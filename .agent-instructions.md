# QUORIX UI Rules For AI Agents

## 1. Vai trò của tài liệu này

Bạn là AI Agent đang sinh UI cho dự án này. Bạn phải coi tài liệu này là luật bắt buộc, không phải gợi ý.

Mục tiêu:

- Chỉ dùng đúng API đã tồn tại trong Quorix UI.
- Không bẻ hệ thống bằng Tailwind, CSS tự chế, hay class bịa ra.
- Giữ chuẩn truy cập WCAG 2.1 AA.

## 2. Nguồn sự thật (Source of Truth)

Khi có mâu thuẫn, ưu tiên theo thứ tự sau:

1. `css/index.css` và thứ tự import thật của nó.
2. Token và class trong `css/colors.css`, `css/metrics.css`, `css/typography.css`.
3. Component classes trong `css/primitives.css`, `css/forms.css`, `css/layout.css`, `css/components.css`, `css/editorial.css`, `css/animations.css`, `css/utilities.css`, `css/icons.css`.
4. Hành vi JS trong `js/quorix.ts`.
5. `docs/visualization.html` chỉ là demo. Nếu demo dùng class/API không tồn tại trong CSS/JS shipped thì bỏ qua demo, không học theo demo.

## 3. Quy tắc bất khả thương lượng

- KHÔNG dùng Tailwind.
- KHÔNG dùng Bootstrap.
- KHÔNG thêm `<style>` mới, file CSS mới, CSS module mới, styled-components mới, hay class CSS tự chế.
- KHÔNG bịa thêm class `qx-*` chưa tồn tại trong `css/*.css`.
- KHÔNG dựa vào class demo không tồn tại thật.
- KHÔNG dùng `data-qx-*` ngoài các API thật sự có trong `js/quorix.ts`.
- Mọi UI mới phải giữ contrast đạt WCAG 2.1 AA, focus nhìn thấy rõ, và không phá keyboard navigation.

## 4. Luật màu sắc

### 4.1. Phân biệt 2 nhóm màu

Quorix có 2 tầng màu:

- Brand / Accent tokens: dùng cho nền, fill, border nhấn mạnh, trạng thái active.
- Adaptive Contrast text tokens: dùng cho chữ có màu trên nền sáng/tối nhưng vẫn giữ contrast an toàn.

- Dùng `--qx-brand-*` cho nền nhấn mạnh, nút chính, border active, badge fill, icon accent.
- Dùng `--qx-text-*` cho text màu cần giữ contrast trên nền trung tính/surface.
- Dùng `--qx-text-on-accent` khi chữ nằm trên nền brand đậm.
- Dùng `--qx-text-on-yellow` khi chữ nằm trên nền vàng.
- Không tự chọn hex.
- Không dùng màu chỉ để “đẹp”; màu phải mang nghĩa semantic.

### 4.5. Cảnh báo rất quan trọng về Adaptive Contrast

`colors.css` định nghĩa logic adaptive contrast đúng qua `--qx-text-blue|red|green|yellow`.

Nhưng `utilities.css` được import sau và override các class sau bằng brand color với `!important`:

- `.qx-text-primary`
- `.qx-text-success`
- `.qx-text-danger`
- `.qx-text-warning`

Vì vậy:

- Không giả định `.qx-text-primary` luôn là adaptive-contrast text utility trong bundle cuối.
- Nếu cần chữ màu có contrast nghiêm ngặt, ưu tiên component đã được thiết kế sẵn theo token, hoặc dùng text mặc định `--qx-text-main` / `--qx-text-muted`.
- Không phát minh class `.qx-text-blue`, `.qx-text-red`, `.qx-text-green`, `.qx-text-yellow`. Chúng không tồn tại trong CSS shipped.

## 6. Declarative JS API (`data-qx-*`)

Chỉ có các API khai báo sau được xác nhận trong `js/quorix.ts`.

### 6.1. Tabs

Trigger:

- Phải có `data-qx-tab="<pane-id>"`
- Phải có `data-qx-tab-group="<group-name>"`
- Tab active dùng thêm class `.is-active`

Pane:

- Phải có `id="<pane-id>"`
- Phải có `data-qx-pane-group="<group-name>"`
- Pane không active phải ẩn bằng `.qx-d-none`

Hành vi:

- Click vào tab sẽ xóa `.is-active` khỏi mọi tab cùng group.
- Click vào tab sẽ thêm `.qx-d-none` cho mọi pane cùng group.
- Pane target sẽ được bỏ `.qx-d-none`.

Mẫu chuẩn:

```html
<nav class="qx-tabs">
  <button class="qx-tab is-active" data-qx-tab="overview" data-qx-tab-group="product-tabs">Overview</button>
  <button class="qx-tab" data-qx-tab="specs" data-qx-tab-group="product-tabs">Specs</button>
</nav>

<section id="overview" data-qx-pane-group="product-tabs">...</section>
<section id="specs" data-qx-pane-group="product-tabs" class="qx-d-none">...</section>
```

### 6.2. Modals

Open trigger:

- Dùng `data-qx-modal-target="<modal-id>"`

Modal wrapper:

- Phải có `id="<modal-id>"`
- Phải dùng `.qx-dialog-backdrop`
- Mặc định nên ẩn bằng `.qx-d-none`

Modal panel:

- Dùng `.qx-dialog-panel`

Close trigger:

- Dùng `data-qx-dismiss`
- Hoặc `data-qx-dismiss="#selector"`

Hành vi:

- JS tự mở modal, bỏ `.qx-d-none`, khóa scroll body.
- JS tự gán `role="dialog"` và `aria-modal="true"` khi mở.
- `Escape` sẽ đóng modal đang mở.
- Focus trap hoạt động trong modal.
- Khi đóng modal, focus quay lại phần tử đã mở modal trước đó.

Mẫu chuẩn:

```html
<button class="qx-btn qx-btn-primary" data-qx-modal-target="delete-modal">Delete</button>

<div id="delete-modal" class="qx-dialog-backdrop qx-d-none" aria-labelledby="delete-title">
  <div class="qx-dialog-panel">
    <div class="qx-dialog-header">
      <h2 id="delete-title">Confirm deletion</h2>
      <button class="qx-dialog-close" data-qx-dismiss aria-label="Close">✕</button>
    </div>
    <p class="qx-text-body">This action cannot be undone.</p>
    <div class="qx-d-flex qx-justify-end qx-gap-3 qx-mt-6">
      <button class="qx-btn qx-btn-ghost" data-qx-dismiss>Cancel</button>
      <button class="qx-btn qx-btn-destructive" data-qx-dismiss>Delete</button>
    </div>
  </div>
</div>
```

### 6.3. Theme

Theme toggle:

- Dùng `data-qx-theme-toggle` trên button hoặc control bất kỳ.

Theme state:

- Root state thực nằm ở `document.documentElement[data-theme="light|dark"]`
- JS lưu state vào `localStorage["qx-theme"]`
- Nếu chưa có state lưu, JS đọc `prefers-color-scheme`

Mẫu chuẩn:

```html
<html data-theme="light">
  <button class="qx-btn qx-btn-ghost qx-btn-sm" data-qx-theme-toggle>Toggle theme</button>
</html>
```

Quy tắc:

- Không phát minh `data-qx-theme="dark"`.
- Không phát minh `data-qx-theme-set`.
- Nếu cần ép theme cố định, chỉnh `data-theme` ở root; nếu cần toggle, dùng `data-qx-theme-toggle`.

### 6.4. API không tồn tại nhưng demo có nhắc tới

Các data attribute sau không được hỗ trợ trong `js/quorix.ts`:

- `data-qx-tooltip`
- `data-qx-tooltip-pos`

Không dùng chúng như JS API chính thức.

## 7. Accessibility Rules

- Tất cả text phải đạt WCAG 2.1 AA.
- Không dùng brand color sáng làm body text trên nền sáng nếu chưa được adaptive-contrast hóa.
- Mọi button, input, checkbox, radio, switch phải giữ focus-visible.
- Modal phải có tiêu đề, nên có `aria-labelledby`, và phải chứa phần tử focus được.
- Toast error phải được coi là ưu tiên cao; toast info/success là polite.
- Tôn trọng `prefers-reduced-motion`.
- Không xóa outline/focus ring mặc định của Quorix.

## 9. Quy tắc về quorix-overrides.css và deficiencies.md

### 9.1. Bắt buộc có comment link khi thêm vào quorix-overrides.css

Bất kỳ đoạn CSS nào được thêm vào `assets/css/quorix-overrides.css` **bắt buộc** phải có một dòng comment ngay phía trên, link trực tiếp đến mục tương ứng trong `.grow/deficiencies.md`.

Ví dụ:
```css
/* deficiencies.md § 4. Scroll-to-top Button (FAB) */
#scroll-to-top { ... }
```

Không được thêm CSS vào `quorix-overrides.css` mà không có comment này. Nếu không tìm thấy mục phù hợp trong `deficiencies.md`, phải tạo mục mới trước, rồi mới thêm CSS.

### 9.2. Mọi CSS tự chế đều phải vào quorix-overrides.css

Bất kỳ đoạn CSS nào không thuộc `@quorix/ui` — dù là fix nhỏ, layout riêng, hay component tự chế — **bắt buộc** phải được đặt trong `assets/css/quorix-overrides.css`.

Tuyệt đối không được:
- Viết `<style>` inline trong bất kỳ template HTML nào.
- Tạo file CSS mới ngoài `quorix-overrides.css`.
- Thêm `style="..."` attribute trực tiếp vào element (trừ trường hợp giá trị là CSS custom property động từ Hugo template).

### 9.3. Được phép dùng class tự chế từ quorix-overrides.css

Nếu một thiết kế yêu cầu các tính năng đã được liệt kê trong `.grow/deficiencies.md` (tức là Quorix UI chưa cung cấp API chính thức), bạn **được phép** sử dụng class Quorix UI chuẩn kết hợp với các class/ID tự chế có sẵn trong `assets/css/quorix-overrides.css`.

Điều kiện:
- Class/ID tự chế đó **phải đã tồn tại** trong `quorix-overrides.css` — không được bịa thêm class mới ngoài file đó.
- Nếu cần thêm class mới, phải thêm vào `quorix-overrides.css` kèm comment link deficiencies, không được inline vào template hay tạo `<style>` mới.

## 10. Default implementation checklist for every AI-generated UI

- Chỉ dùng class `.qx-*` đã có thật trong `css/*.css`.
- Ưu tiên ghép từ primitives + utilities thay vì viết CSS mới.
- Nếu cần tabs, modals, theme toggle thì dùng đúng `data-qx-*` API ở trên.
- Nếu một ý tưởng UI cần class không tồn tại, phải đổi thiết kế để dùng class hiện có, không được bịa class.
- Nếu một màu không chắc đạt contrast AA, quay về `--qx-text-main`, `--qx-text-muted`, hoặc component variant đã có sẵn.
